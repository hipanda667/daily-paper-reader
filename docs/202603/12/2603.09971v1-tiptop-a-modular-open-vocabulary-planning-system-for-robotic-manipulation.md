## 摘要
我们提出了 TiPToP，这是一个可扩展的模块化系统，它将预训练的视觉基础模型与现有的任务与运动规划器（TAMP）相结合，直接根据输入的 RGB 图像和自然语言指令解决多步操作任务。我们的系统旨在简单易用：它可以在不到一小时的时间内安装并在标准的 DROID 设置上运行，并能以极小的代价适配到新的机器人本体。我们在仿真和现实世界的 28 个桌面操作任务中评估了 TiPToP（该系统不需要任何机器人数据），发现其表现与 $π_{0.5}\text{-DROID}$ 相当或更优，后者是在 350 小时特定本体演示数据上微调的视觉-语言-动作（VLA）模型。TiPToP 的模块化架构使我们能够在组件层面分析系统的故障模式。我们分析了 173 次试验的评估结果，并确定了改进方向。我们开源了 TiPToP，以进一步推动模块化操作系统以及学习与规划之间更紧密集成的研究。项目网站和代码：https://tiptop-robot.github.io

## Abstract
We present TiPToP, an extensible modular system that combines pretrained vision foundation models with an existing Task and Motion Planner (TAMP) to solve multi-step manipulation tasks directly from input RGB images and natural-language instructions. Our system aims to be simple and easy-to-use: it can be installed and run on a standard DROID setup in under one hour and adapted to new embodiments with minimal effort. We evaluate TiPToP -- which requires zero robot data -- over 28 tabletop manipulation tasks in simulation and the real world and find it matches or outperforms $π_{0.5}\text{-DROID}$, a vision-language-action (VLA) model fine-tuned on 350 hours of embodiment-specific demonstrations. TiPToP's modular architecture enables us to analyze the system's failure modes at the component level. We analyze results from an evaluation of 173 trials and identify directions for improvement. We release TiPToP open-source to further research on modular manipulation systems and tighter integration between learning and planning. Project website and code: https://tiptop-robot.github.io

### 1. 核心问题与整体含义（研究动机和背景）
论文旨在解决机器人操作中“开箱即用”的通用性问题。目前的视觉-语言-动作（VLA）模型（如 $\pi_0$、OpenVLA）虽然能处理自然语言和图像，但通常需要海量的机器人特定演示数据进行微调，且难以诊断故障原因。传统的任务与运动规划（TAMP）虽然逻辑严密，但往往依赖预定义的几何模型，缺乏对开放词汇和复杂场景的感知能力。
**TiPToP** 的核心动机是构建一个**模块化、无需机器人训练数据、且能直接处理像素输入和自然语言指令**的系统。它通过组合现有的视觉基础模型与高效的 GPU 并行规划器，实现了在多种机器人本体上的快速部署。

### 2. 方法论
TiPToP 将任务分解为三个核心模块：
*   **感知模块（Perception）：**
    *   **3D 视觉分支：** 使用 *FoundationStereo* 从双目图像估计深度，生成 3D 点云；使用 *M2T2* 在点云上预测 6-DoF 抓取位姿。
    *   **语义分支：** 使用 *Gemini 1.5 Pro* (VLM) 进行目标检测并根据指令生成符号化目标（如 `On(A, B)`）；使用 *SAM-2* 生成精确的分割掩码。
    *   **融合：** 将点云、掩码和抓取位姿结合，通过凸包（Convex Hull）重建每个物体的 3D 网格模型。
*   **规划模块（Planning）：**
    *   使用 **cuTAMP**（一种 GPU 并行的 TAMP 算法）。它首先枚举任务计划骨架（如：移动->抓取->放置），然后利用 GPU 加速的微分优化来寻找满足碰撞避免、运动学可行性和放置稳定性的连续参数（抓取点、放置点、轨迹）。
*   **执行模块（Execution）：**
    *   采用自定义的关节阻抗控制器，精确跟踪规划出的时间参数化轨迹。该过程目前是**开环**的，即规划完成后一次性执行。

### 3. 实验设计
*   **场景与任务：** 涵盖了 28 个桌面操作任务，分为四类：简单任务、带干扰物的任务、语义推理任务（如“把最大的玩具放进盘子”）和多步规划任务（如移动障碍物以抓取目标）。
*   **实验环境：** 包括 IsaacSim 仿真环境、内部 DROID 硬件平台、以及由外部团队独立操作的 DROID 平台（验证易用性）。此外还测试了 UR5e 和 WidowX AI 机器人。
*   **Benchmark 与对比方法：** 主要对比对象是 **$\pi_{0.5}\text{-DROID}$**，这是一个在 350 小时特定机器人演示数据上微调过的最先进 VLA 模型。
*   **评估指标：** 成功率（Success Rate）和任务进度（Task Progress）。

### 4. 资源与算力
*   **训练算力：** **零训练需求**。TiPToP 直接使用预训练的基础模型，不需要任何机器人本体的演示数据。
*   **推理算力：** 实验使用了 NVIDIA L4（仿真）、RTX 3080 Laptop（外部评估）和 RTX 4090（内部评估）。
*   **部署耗时：** 论文强调系统安装和校准在 1 小时内即可完成，适配新机器人仅需数小时。

### 5. 实验数量与充分性
*   **实验规模：** 进行了 165 次正式评估试验。为了深入分析，还额外进行了 173 次故障分析试验。
*   **客观性：** 引入了**外部评估团队**进行独立测试，证明了系统在非开发者手中的鲁棒性和易部署性。
*   **充分性：** 实验覆盖了从仿真到现实、从单步到长程、从简单指令到复杂语义的多种维度，对比实验设计公平（相同输入、相同起始位姿）。

### 6. 主要结论与发现
*   **性能超越：** 在复杂语义和多步规划任务中，TiPToP 显著优于经过微调的 $\pi_{0.5}\text{-DROID}$。
*   **效率更高：** TiPToP 的执行速度通常比 VLA 模型快一倍，因为它规划的是时间最优轨迹，而 VLA 往往存在“闲置”或反复尝试的行为。
*   **模块化优势：** 能够清晰地定位失败原因（如 56% 的失败源于抓取不稳定，24% 源于感知重建误差）。
*   **跨平台通用：** 证明了该系统可以轻松迁移到 UR5e 等不同动力学结构的机器人上。

### 7. 优点
*   **零样本学习：** 无需昂贵的机器人数据采集，利用互联网规模的视觉/语言知识。
*   **可解释性与可调试性：** 模块化设计允许开发者针对特定组件（如深度估计或抓取预测）进行独立升级。
*   **强大的推理能力：** 结合 VLM 和 TAMP，能够处理需要常识推理的长程任务（如“清理白板并收纳工具”）。
*   **开源贡献：** 提供了完整的代码库和仿真环境，降低了 TAMP 研究的门槛。

### 8. 不足与局限
*   **开环执行：** 缺乏实时视觉反馈。如果物体在执行过程中发生滑动或位移，系统无法自动纠正（这是与 VLA 相比的主要劣势）。
*   **单视角感知：** 仅依靠单次拍摄的视角，容易受到遮挡影响，且物体背面的几何形状靠凸包近似，不够精确。
*   **感知精度瓶颈：** 凸包近似无法处理凹面物体（如杯子内部或弯曲的香蕉），会导致碰撞检查错误。
*   **动态环境适应性：** 假设环境在规划后是静态的，无法处理移动中的物体。
